# Security Policy

Saturnis LLC takes the security of Cassini seriously. We appreciate the community's efforts in responsibly disclosing vulnerabilities and helping us keep the platform safe for all users.

## Supported Versions

Security updates are provided for the **latest release** only. We recommend always running the most recent version of Cassini.

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| Older   | No        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, email us at **[security@saturnis.io](mailto:security@saturnis.io)** with the following information:

- **Description** of the vulnerability
- **Steps to reproduce**, including any relevant configuration or environment details
- **Impact assessment** -- what an attacker could achieve by exploiting this issue
- **Affected component** (backend, frontend, bridge, or infrastructure)
- Any **proof-of-concept** code or screenshots, if available

### What to Expect

| Step | Timeline |
|------|----------|
| Acknowledgment of your report | Within 48 hours |
| Initial assessment and severity classification | Within 7 days |
| Fix development and release | Target within 90 days |

We will keep you informed of progress throughout the process. If we need additional information, we will reach out to you directly.

## Disclosure Policy

- **Do not** publicly disclose the vulnerability until a fix has been released.
- We will coordinate with you on disclosure timing and provide advance notice before the fix is published.
- We aim to release fixes as quickly as possible, with critical vulnerabilities prioritized accordingly.

## Credit

We believe in recognizing the contributions of security researchers. With your permission, we will credit you in the release notes for any confirmed vulnerability you report. Let us know your preferred name or handle when submitting your report.

## Out of Scope

The following are generally outside the scope of this policy:

- **Social engineering** attacks against Saturnis employees or community members
- **Denial of service** (DoS/DDoS) attacks
- **Vulnerabilities in third-party dependencies** -- please report these to the upstream project directly. If a dependency vulnerability affects Cassini specifically, let us know and we will prioritize updating.
- **Issues requiring physical access** to a server or device
- **Automated scanner output** without a demonstrated, exploitable vulnerability

## Bug Bounty

We do not currently operate a formal bug bounty program. However, we deeply appreciate responsible disclosure and will recognize contributors who help improve Cassini's security.
