# v0.3.0 Summary

> **Archive location**: `.planning/archive/v0.3.0/` (18 phase directories, ~100 planning documents)
> **Period**: v0.1.0 through v0.3.0 (project inception through 2026-02-07)

---

## What Was Built

v0.3.0 established Cassini as a functional SPC platform with core charting, multi-plant architecture, user management, and industrial connectivity.

### v0.1.0 -- Core SPC Foundation
- Variable control charts (X-bar/R, X-bar/S, I-MR)
- All 8 Nelson Rules with zone classification
- Plant/Line/Station/Characteristic hierarchy model
- Report generation (PDF/PNG/SVG)
- Kiosk mode (full-screen auto-rotate)
- Light/dark theming with custom branding

### v0.2.0 -- Plant-Scoped Configuration
- Plant model with CRUD operations
- Per-plant data isolation
- Plant-scoped characteristic configuration
- Multi-plant hierarchy management

### v0.3.0 -- User Management & Connectivity
- **User management**: 4-tier RBAC (operator/supervisor/engineer/admin), per-plant roles via `user_plant_role` join table, JWT auth (15min access + 7d refresh cookie)
- **MQTT/SparkplugB connectivity**: aiomqtt integration, topic discovery, tag mapping to characteristics, real-time data ingestion
- **Charting enhancements**: Annotations (click-to-annotate), time-axis mode, ECharts 6 migration from Chart.js
- **Reporting**: PDF export, chart snapshots, configurable templates
- **UI polish**: Enterprise UI overhaul, dashboard UX improvements

## Codebase at v0.3.0 End

- Backend: ~19 models, ~15 routers (~130 endpoints), 15 Alembic migrations
- Frontend: ~70 components, 10 pages, ECharts 6 (canvas)
- Auth: JWT + 4-tier RBAC, per-plant roles
- Data: SQLite (dev), MQTT real-time ingestion
- Charts: X-bar/R, X-bar/S, I-MR (variable only)

## Key Milestones

- First working SPC chart with Nelson Rules
- ECharts migration (from Chart.js) -- significant rendering improvement
- MQTT real-time data pipeline operational
- Multi-plant architecture with data isolation
- Kiosk mode for shop floor displays
