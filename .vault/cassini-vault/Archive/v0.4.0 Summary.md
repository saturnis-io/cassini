# v0.4.0 Summary -- Enterprise

> **Archive location**: `.planning/archive/v0.4.0/` (workstreams, designs, skeptic reports, milestone docs)

## What Was Delivered

v0.4.0 transformed Cassini from a basic variable-data SPC tool into a competitive enterprise platform. Across 6 workstreams and 4 sprints, the codebase grew from ~130 to 149+ endpoints, 20 to 30 Alembic migrations, and the [[Strategy/Competitive Analysis 2026|Tier 1 scorecard]] went from 2/8 to 7/8.

---

## Workstream 1: Multi-Database Support (WS-1)

Full database management for SQLite, PostgreSQL, MySQL, and MSSQL.

- **Dialect abstraction layer** (`db/dialects.py`): Automatic detection, connection URL parsing, per-dialect pool tuning
- **Encrypted credential storage**: `db_config.json` with Fernet encryption, key in `.db_encryption_key`
- **Database admin API**: 7 endpoints (config, test, status, backup, vacuum, migrations)
- **Frontend admin UI**: DatabaseSettings, ConnectionForm, MigrationStatus, MaintenancePanel
- **Migration 016**: Full portability rewrite of all prior migrations

**Relates to**: [[Features/Multi-Database]]

---

## Workstream 2: OPC-UA Integration (WS-2) -- 4 Phases

Triple-protocol industrial connectivity (MQTT + OPC-UA + later RS-232).

- **Phase 1**: Polymorphic JTI data model (`DataSource` base + `MQTTDataSource` + `OPCUADataSource`)
- **Phase 2**: asyncua client with reconnect, OPCUAServer model, 12 API endpoints, node browsing
- **Phase 3**: OPCUAProvider with subscription-based data ingestion, shared SubgroupBuffer
- **Phase 4**: Unified 4-tab Connectivity Hub UI (28 components), protocol registry

**Migrations**: 017 (JTI data sources), 018 (OPC-UA server table)

**Relates to**: [[Features/OPC-UA Integration]]

---

## Workstream 3: MQTT Outbound Publishing (WS-3)

Publish SPC analysis results via MQTT.

- MQTTPublisher (Event Bus subscriber pattern)
- UNS-aware topic naming: `openspc/{plant}/{hierarchy}/{char}/{event_type}`
- 4 event types: SampleProcessed, ViolationCreated, ViolationAcknowledged, ControlLimitsUpdated
- Dual format: JSON + SparkplugB
- Per-characteristic rate limiting

**Relates to**: [[Features/MQTT Connectivity]]

---

## Workstream 4: Architecture Review (WS-4) -- 3 Phases

Enterprise-readiness audit and hardening.

- **Phase 1 (Security)**: RBAC plant-scoping fix, forced password change, ValueError sanitization, proactive token refresh
- **Phase 2 (Code Quality)**: SlowAPI rate limiting, zone dedup, server-side filtering, optimistic updates on 4 mutations
- **Phase 3 (Production)**: structlog migration, auth/RBAC test suites, test environment scripts

**Skeptic audit**: 38 findings, 16 fixed, 4 partially, 18 outstanding (mostly LOW).

---

## Workstream 5: Schema Hardening (WS-5)

Database integrity and production readiness.

- CASCADE foreign keys on all child relationships
- Timezone-aware datetimes across all models
- Broker password encryption at rest (Fernet)
- `Violation.char_id` denormalized column
- Composite indexes for common query patterns
- Plant-scoped unique name constraints

**Migration 020**

---

## Workstream 6: Records Retention (WS-6)

Configurable data lifecycle management.

- `retention_policy` table with 3-scope inheritance (global/hierarchy/characteristic)
- Walk hierarchy tree from characteristic up to plant default
- 10 API endpoints + purge engine (24h cycle, batch deletion, per-plant history)
- Retention UI with policy tab, overrides tree browser, activity log

**Migrations**: 021, 022

**Relates to**: [[Features/Records Retention]]
