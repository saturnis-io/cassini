# Show Your Work

A trust feature for regulated industries that lets users click any statistical value to see the exact formula, step-by-step computation, and inputs -- generated by the same engine that produced the number.

## Problem Statement

In regulated industries (automotive, aerospace, medical), auditors and engineers need to trust that computed statistical values are correct. Cassini displays results (Cpk, GRR%, Kappa, etc.) as opaque numbers -- users must trust the system or manually verify with a calculator. "Show Your Work" makes every value transparent and auditable.

## Key Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Initial scope | Capability + MSA metrics (Tier 2) | Highest regulatory trust ROI -- auditors question these most |
| Display format | Right slide-out panel | Stays open while navigating between values |
| Data source | Backend-computed (same code path) | Zero drift risk -- explanation generated by the same engine function |
| Toggle UX | Persistent header icon, localStorage | Survives navigation |
| Formula rendering | KaTeX (~28KB gzipped) | Professional math typesetting for regulatory expectations |
| Architecture | Inline explain mode (ExplanationCollector pattern) | No duplicated methods -- same code path with optional step capture |
| Caching | React Query, lazy-load on click, staleTime: Infinity | No wasted API calls; cache busted when underlying data changes |

## Architecture

### Backend: ExplanationCollector Pattern

Existing compute methods gain an optional `collector` parameter. When provided, intermediate steps are captured alongside the normal computation. When `None` (the default), zero overhead -- just `if collector:` checks (nanosecond cost vs the numpy/scipy operations they surround).

```python
class ExplanationCollector:
    def step(self, label, formula_latex, substitution_latex, result, note=None): ...
    def input(self, name, value): ...
    def warn(self, message): ...
```

### Backend: Explain API

Single endpoint: `GET /api/v1/explain/{metric_type}/{characteristic_id}`

Response includes: metric name, display name, value, general formula (KaTeX), step-by-step breakdown, named inputs, AIAG/ISO citation, method used, and warnings.

### Two Data Modes (Critical)

1. **With chart options** (`start_date`/`end_date`/`limit`): Uses subgroup means + sigma from means -- matches dashboard `quickStats`
2. **Without chart options**: Flattens individual measurements + uses `stored_sigma` (R-bar/d2) -- matches capability GET endpoint / CapabilityCard

### Citation Registry

Static mapping of standard references per metric family:
- **Capability**: AIAG SPC Manual, 2nd Edition, Chapter 3
- **MSA**: AIAG MSA Manual, 4th Edition
- **Normality**: ISO 11462-1:2001
- **Attribute MSA**: AIAG MSA Manual, 4th Edition, Chapter 5

### Frontend Architecture

- **`useShowYourWorkStore`** (Zustand): Global toggle (persisted to localStorage) + active metric state
- **`<Explainable>`**: Wrapper component -- when mode is off, renders children unchanged (zero overhead); when on, adds dotted underline + click handler
- **`<ExplanationPanel>`**: Slide-out drawer (360px, z-40) with KaTeX-rendered formulas, numbered steps, inputs table, citation block, and warnings
- **Header toggle**: Sigma icon in AppHeader.tsx

### Metric Coverage

| Phase | Components | Values Wrapped |
|---|---|---|
| Phase 1 (Foundation + Capability) | CapabilityCard | Cp, Cpk, Pp, Ppk, Cpm (~5) |
| Phase 2 (MSA) | MSAResults, AttributeMSAResults, DistributionAnalysis | GRR%, ndc, EV, AV, Kappa (~26) |
| Phase 3 (Future) | Control chart limits | UCL, LCL, CL, sigma |
| Phase 4 (Future) | Advanced analytics | Anomaly, Nelson rules, short-run |

## Components Built

### Backend
- `core/explain.py` -- ExplanationCollector class, Citation registry
- `api/schemas/explain.py` -- Pydantic response schemas
- `api/v1/explain.py` -- Explain API router
- `core/capability.py` -- Instrumented with collector (Cp, Cpk, Pp, Ppk, Cpm steps)

### Frontend
- `stores/showYourWorkStore.ts` -- Zustand store with persist middleware
- `components/Explainable.tsx` -- Wrapper primitive
- `components/ExplanationPanel.tsx` -- Slide-out with KaTeX rendering
- `api/explain.api.ts` + `api/hooks/useExplanation.ts` -- API namespace and hook
- Header toggle + Layout mount

## Non-Goals

- No database changes -- explanations are computed on-the-fly
- No export/print in initial scope
- No per-user permissions -- any authenticated user with plant access can use it
- No AI-generated explanations -- formulas only
